# Documentation CapEvents

Ce dossier contient la documentation technique et fonctionnelle du projet **CapEvents**.

## Structure du dossier

### `sprints/`
Contient le détail du projet par sprint :
- `sprint-1.md`
- `sprint-2.md`
- `sprint-3.md`

### `diagrams/`
Contient les diagrammes du projet au format Mermaid :
- architecture
- déploiement
- authentification
- cas d’utilisation
- cycle de vie des événements
- inscriptions, invitations, RSVP
- propositions et validations
- feedback, points, notifications
- analytics
- modèle de données
- cartographie des routes et des rôles

## Organisation recommandée

La documentation est séparée en deux niveaux :
- **documentation par sprint** pour expliquer l’évolution du projet
- **documentation par diagrammes** pour représenter l’architecture, les flux métier et les interactions système

## Objectifs de cette documentation

Cette documentation a pour objectif de :
- centraliser les informations du projet
- faciliter la compréhension globale de la plateforme
- conserver une trace claire de l’évolution fonctionnelle et technique
- préparer les éléments réutilisables dans le mémoire et la soutenance

- Voir le détail des sprints dans `sprints/`
